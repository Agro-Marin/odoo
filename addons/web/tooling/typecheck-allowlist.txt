# Type-check allowlist — files that MUST stay `tsc --noEmit` clean.
#
# Paths are relative to the Odoo community (core) repo root — the same root
# tooling/enable.sh installs jsconfig.json + node_modules into, and the same
# base the jsconfig `@web/*` mapping uses (addons/web/static/src/*).
#
# The CI gate (tooling/typecheck-ci.sh) runs tsc over each file below plus its
# transitive import graph, then FAILS if any error is attributed to a file in
# this list. Errors in imported-but-unlisted files are tolerated — they are the
# codebase's pre-existing type debt, ratcheted down file-by-file.
#
# HOW TO ADD A FILE:
#   1. Eliminate its `/** @type {any} */` escapes (see machine_doc_v1/
#      JSDOC_TYPE_TIGHTENING.md for the recipe).
#   2. Run: bash tooling/typecheck-ci.sh --probe addons/web/static/src/<path>
#      It must report 0 errors for that path.
#   3. Add the path here.
#
# IMPORTANT: "any-escapes removed" is NOT the same as "tsc-clean". Removing a
# cast often REVEALS errors the `any` was suppressing. Always verify with the
# --probe command before adding — do not trust a file's status from a doc.
# (Measured 2026-05-28: record.js had 0 any-escapes per the doc but 32 real
# tsc errors; relational_model.js 29; action_service.js 7. Only rpc.js was
# genuinely clean among the documented "Done" seam files.)
